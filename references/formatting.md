# Formatting & Conversion Reference

## Universal Formatting Rules
- H1 for chapter titles only (one per chapter)
- H2/H3 for sections — never H1 for sections
- No tab indentation — use paragraph styles
- Images: embedded, minimum DPI per output type (see SKILL.md)
- Links: absolute URLs only (relative links break in EPUB readers)
- No custom headers/footers — platforms inject their own
- No page numbers in content — auto-generated by reader apps

## EPUB Tool Chain (Standard Ebooks)

### Markdown → EPUB via Pandoc (recommended)
```bash
# Basic conversion
pandoc manuscript.md -o ebook.epub \
  --epub-cover-image=cover.jpg \
  --epub-metadata=metadata.xml \
  --toc --toc-depth=2

# With custom CSS styling
pandoc manuscript.md -o ebook.epub \
  --epub-cover-image=cover.jpg \
  --css=epub-styles.css \
  --epub-metadata=metadata.xml \
  --toc --toc-depth=2 \
  --highlight-style=kate

# Multi-chapter (separate files, ordered)
pandoc intro.md ch01.md ch02.md conclusion.md -o ebook.epub \
  --epub-cover-image=cover.jpg \
  --epub-metadata=metadata.xml \
  --toc --toc-depth=2
```

### Python: Markdown → EPUB (programmatic, inspired by smerchek/claude-epub-skill, MIT)
```python
# scripts/build_epub.py
# Requires: pip install ebooklib markdown beautifulsoup4

import os
import markdown
from ebooklib import epub
from bs4 import BeautifulSoup

def build_epub(title, author, chapters_dir, output_path, cover_path=None):
    book = epub.EpubBook()
    book.set_identifier(f"id_{title.replace(' ', '_').lower()}")
    book.set_title(title)
    book.set_language("en")
    book.add_author(author)

    chapters = []
    spine = ["nav"]

    # Load chapters in order
    for fname in sorted(os.listdir(chapters_dir)):
        if not fname.endswith(".md"):
            continue
        with open(os.path.join(chapters_dir, fname)) as f:
            content = markdown.markdown(f.read())
        soup = BeautifulSoup(content, "html.parser")
        chapter_title = soup.find("h1").text if soup.find("h1") else fname.replace(".md", "")

        chapter = epub.EpubHtml(
            title=chapter_title,
            file_name=fname.replace(".md", ".xhtml"),
            lang="en"
        )
        chapter.content = f"<html><body>{content}</body></html>"
        book.add_item(chapter)
        chapters.append(chapter)
        spine.append(chapter)

    # Cover
    if cover_path:
        with open(cover_path, "rb") as f:
            book.set_cover("cover.jpg", f.read())

    # Navigation
    book.toc = tuple(epub.Link(c.file_name, c.title, c.file_name) for c in chapters)
    book.add_item(epub.EpubNcx())
    book.add_item(epub.EpubNav())
    book.spine = spine

    epub.write_epub(output_path, book)
    print(f"EPUB written to {output_path}")

# Usage:
# build_epub("My Book", "Author Name", "./chapters/", "output.epub", "cover.jpg")
```

## HTML → PDF Conversion (Designed Ebooks)

For visually designed ebooks using fixed `.page` containers, custom fonts,
and complex layouts — the **Puppeteer screenshot workflow** is the only
reliable method for pixel-perfect output.

### ⚠️ Critical Warnings
- **Never use WeasyPrint** for fixed `.page` HTML — breaks CSS positioning, ignores custom fonts
- **Never use `page.pdf()`** (Puppeteer's native PDF export) — clips shadows, loses backgrounds, corrupts multi-column layouts
- The screenshot workflow is slower but renders identically to what the browser shows

### The 6-Step Puppeteer Screenshot Workflow

Reference implementation on VPS: `/var/www/geolab-ebooks/html-to-pdf-final.js`

**Step 1 — Launch with correct viewport**
```javascript
const puppeteer = require("puppeteer");
const { PDFDocument } = require("pdf-lib");
const fs = require("fs");
const path = require("path");

const browser = await puppeteer.launch({ headless: "new" });
const page = await browser.newPage();
// Match your .page container dimensions exactly
await page.setViewport({ width: 794, height: 1123, deviceScaleFactor: 2 });
```

**Step 2 — Load HTML and wait for all assets**
```javascript
await page.goto(`file://${path.resolve(htmlFile)}`, {
  waitUntil: "networkidle0"   // waits for fonts, images, all network requests
});
// Extra wait for web fonts if needed
await page.waitForTimeout(1000);
```

**Step 3 — Hide interactive elements, add callout boxes**
```javascript
await page.addStyleTag({ content: `
  .interactive-widget,
  .hover-tooltip,
  video, iframe,
  [data-interactive],
  .quiz-container { display: none !important; }
` });
```

For each hidden interactive element, the HTML should already contain a callout:
```html
<div class="visit-site-callout">
  🔗 Interactive version at <strong>yoursite.com/tool-name</strong>
</div>
```
These become lead funnel touchpoints — use tracked URLs.

**Step 4 — Screenshot each page individually**
```javascript
const pages = await page.$$(".page");
const images = [];

for (let i = 0; i < pages.length; i++) {
  const img = await pages[i].screenshot({
    type: "png",
    omitBackground: false    // preserve backgrounds — critical for designed ebooks
  });
  images.push(img);
  console.log(`Captured page ${i + 1}/${pages.length}`);
}

await browser.close();
```

**Step 5 — Combine into PDF**
```javascript
const pdfDoc = await PDFDocument.create();

for (const imgBuffer of images) {
  const img = await pdfDoc.embedPng(imgBuffer);
  const pdfPage = pdfDoc.addPage([img.width / 2, img.height / 2]); // divide by deviceScaleFactor
  pdfPage.drawImage(img, {
    x: 0, y: 0,
    width: img.width / 2,
    height: img.height / 2
  });
}

const pdfBytes = await pdfDoc.save();
fs.writeFileSync(outputPath, pdfBytes);
console.log(`PDF written: ${outputPath}`);
```

**Step 6 — Verify**
```bash
ls -lh output.pdf          # typical: 2–15MB for a designed ebook
# Open and check: fonts, backgrounds, no clipping, page order correct
```

### Production Workflow (battle-tested, March 2026)

The 6-step workflow above is the basic approach. In production, ebooks with many pages need additional steps:

**Step 2b — Auto-paginate overflowing pages**
Use `auto-paginate.js` on VPS to split overflowing pages by moving children to new `.page` divs. Runs multiple passes. Handles glossary `<dl>` splits at `<dt>` boundaries and `<table>` splits at `<tr>` boundaries.

**Step 2c — Fix orphan headings**
Orphan = heading within 80px of page bottom with no content after it. Move orphan H2/H3/H4 + following siblings to next page. Re-check destination page doesn't overflow.

**Step 2d — Add sequential page numbers**
Add "Page N" to all content page footers. Skip cover and part divider pages (`.part-page`).

**Step 2e — Merge near-empty pages**
After auto-pagination, check for pages with excessive blank space (e.g., only a heading + footer). Merge them back into the previous page if combined content fits within 1123px. This happens when auto-paginator splits too aggressively.

**IMPORTANT — Footer class names vary between ebooks:**
Some use `.pfooter`, others use `.page-footer`. All scripts must query for BOTH: `.pfooter,.page-footer,[class*="pfooter"],[class*="page-footer"]`

**Step 4b — Screenshot slicer tolerance**
`html-to-pdf-final.js` has 50px tolerance: pages overflowing by <50px are single screenshots. Pages >50px are sliced into multiple PDF pages.

**VPS audit scripts:** `final-audit.js`, `full-page-audit.js`, `auto-paginate.js`, `fix-orphans-numbers.js`

**Confirmed failures (DO NOT attempt):**
- `page.addStyleTag()` — ignored by print renderer
- `page.evaluate()` display changes — reverted by print CSS
- Button clicks before `page.pdf()` — state not preserved
- Removing `overflow: hidden` from `.page` — breaks pagination
- WeasyPrint on designed HTML — re-interprets everything

## EPUB Styling Reference

### epub-styles.css (base template)
```css
/* Reset */
body { margin: 0; padding: 0; font-family: Georgia, serif; }

/* Typography */
h1 { font-size: 2em; margin-top: 3em; page-break-before: always; }
h2 { font-size: 1.4em; margin-top: 2em; }
h3 { font-size: 1.2em; }
p { font-size: 1em; line-height: 1.6; margin: 0.8em 0; text-indent: 1.5em; }
p:first-of-type, h2 + p, h3 + p { text-indent: 0; }

/* Images */
img { max-width: 100%; height: auto; display: block; margin: 1em auto; }

/* Code */
code { font-family: monospace; font-size: 0.85em; background: #f5f5f5; padding: 0.1em 0.3em; }
pre { background: #f5f5f5; padding: 1em; overflow-x: auto; }

/* Pull quotes */
blockquote { border-left: 3px solid #333; margin: 1.5em 2em; padding-left: 1em; font-style: italic; }

/* Chapter opener */
.chapter-number { font-size: 0.8em; text-transform: uppercase; letter-spacing: 0.2em; color: #666; }

/* Page break control */
.page-break { page-break-after: always; }
h1, h2 { page-break-after: avoid; }
img, table, pre { page-break-inside: avoid; }
```
